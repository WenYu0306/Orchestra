# Orchestra — 三层 Agent 架构

```
┌──────────────────────────────────────────────────────────────────┐
│          决策层 · DeepSeek Orchestrator                            │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────┐ ┌──────────┐   │
│  │ JD 评分   ││ 分层决策  │ │ 招呼语   │ │假检测│ │ 搜索词   │   │
│  │ (0-100)  │ │高≥80中≥60│ │ 动态生成  │ │      │ │ 动态生成  │   │
│  └──────────┘ └──────────┘ └──────────┘ └──────┘ └──────────┘   │
├──────────────────────────────────────────────────────────────────┤
│          执行层 · 数据获取                                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────────────────┐   │
│  │ XHR 搜索  │ │ 详情重评  │ │ 标签评分  │ │ httpx → DeepSeek │   │
│  │joblist.json│ │card.json │ │          │ │ 共享客户端 50/10  │   │
│  └──────────┘ └──────────┘ └──────────┘ └────────────────────┘   │
├──────────────────────────────────────────────────────────────────┤
│          连接层 · Chrome Bridge                                    │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────────────────┐   │
│  │nodriver  │ │ CDP 协议  │ │ 同步 XHR  │ │ 反爬对抗            │   │
│  │启动Chrome│ │evaluate   │ │ 调BOSS API│ │--disable-blink     │   │
│  └──────────┘ └──────────┘ └──────────┘ └────────────────────┘   │
└──────────────────────────────────────────────────────────────────┘

数据流：Chrome → XHR 搜 joblist.json → 标签评分 → 过滤 → top 30 详情
重评 → 分层 → SSE 推送前端 → 勾选 → XHR 调 friend/add.json 发送
```

## 核心流程

| 步骤 | 说明 |
|------|------|
| 搜索 | 城市×关键词，nodriver Chrome 内跑同步 XHR 调 BOSS joblist.json，30 条/页 |
| 标签评分 | 搜索返回的字段拼成短 JD，DeepSeek 快评 |
| 过滤 | 城市不匹配、薪资不达标、虚假招聘（教育/培训类）跳过 |
| 详情重评 | top 30 用 card.json 拿完整 JD，DeepSeek 重评 |
| 分层 | high(5)/medium(7)/try(8)，优先取详情重评过的 |
| 招呼语 | DeepSeek 基于简历+JD 生成 |
| SSE 推送 | 逐条推给前端，实时显示卡片 |
| 勾选发送 | 前端复选框 + 发送按钮，POST /api/send |
| XHR 发送 | 从按钮 HTML 取 data-url，同步 XHR 调 friend/add.json |

## 组件树

```
App.vue
├── TopBar.vue          Logo + 状态指示器 + 开始/停止
├── ProgressBand.vue    进度条 + 步骤提示
├── StatCards.vue       高分/中分/可试/总数 四张统计卡
├── JobList.vue         排序 + 发送选中按钮 + 卡片列表
│   └── JobCard.vue     公司名 + 职位名 + 分数 + 招呼语 + 复选框
└── BottomBar.vue       底部信息栏
```

## 核心数据

| 指标 | 数值 |
|------|------|
| 搜索方向 | 8-10 个（DeepSeek 动态生成 + 配置预设） |
| 搜索城市 | 北京、长春 |
| 搜索岗位 | 200-350 个/次 |
| 评分方案 | 标签快速初筛 + 详情 API 重评 top 30 |
| 评分区间 | 30-95 分 |
| 最终输出 | 20 个岗位按 5/7/8 分层 |
| 发送 | XHR 调 friend/add.json，BOSS 默认招呼语 |

## 设计理念

- **功能主义。** 进度条和统计卡片数字跳变比动效更重要
- **干扰最小。** 状态灯安静呼吸，遇到问题才变色
- **配色系统。** 浅色基底，靛蓝/蓝/琥珀三层区分
- **毛玻璃面板。** `backdrop-filter: blur(24px)` + 多层阴影
- **Aurora 光晕。** 固定模糊光球，纯装饰
- **框架中立。** Vue 3 单页，SSE 与 UI 解耦
